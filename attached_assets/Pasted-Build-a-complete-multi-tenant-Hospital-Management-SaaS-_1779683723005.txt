Build a complete multi-tenant Hospital Management SaaS web application.

Create a premium, luxury, modern, enterprise-level UI/UX design for a multi-tenant Hospital Management SaaS web application.

use npm package no use pnpm

Design Goal:
Make the UI look like a high-end $1,000,000 SaaS product.
The design should feel premium, clean, modern, trustworthy, medical, enterprise-grade, and suitable for real hospitals and clinics.

Main Goal:
Create one SaaS platform where multiple hospitals/clinics can register and use the same system.
Do not create separate code for every hospital.
Use one codebase, one backend, and one database.
Each hospital must have separate data using hospitalId.

Important:
Do not create a basic admin panel.
Do not use simple Bootstrap-style design.
Do not make it look like a student project.
The UI should look like a premium SaaS product used by real hospitals.

Example:
Hospital A should only see Hospital A doctors, patients, invoices, appointments, reports, and settings.
Hospital B should only see Hospital B data.
Hospital A data should never mix with Hospital B data.
